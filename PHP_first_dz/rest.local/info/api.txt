Разработка API

Работу API обеспечивают скрипты РНР, рекомендуется
обращаться к ним через маршрутизатор (.htaccess)
Это позволит а) скрыть расширение (технологию),
б) убедиться в работоспособности сервера (если не
сработала маршрутизация, то проблемы с сервером)

Разделение по методам запроса происходит из анализа
$_SERVER[ 'REQUEST_METHOD' ]

Передача данных:
данные из URL (Query) попадают в массив $_GET независимо
от метода запроса.
Данные из тела попадают в массив $_POST, но, только если
заголовок Content-Type соответствует html-форме. Иначе,
доступ к телу пакета запроса предоставляет специальный
поток (файл) "php://input". С ним возможны любые файловые
операции чтения, например, file_get_contents( "php://input" ),
возвращающая весь контент в виде строки.
Ответные данные передаются выводящими операциями
echo или print, а также средства разработчика var_dump. 
Статус-код ответа устанавливается
http_response_code( 412 ) ;
заголовки
header( "Key: value" ) ;

Основные параметры запроса и режимов работы собираются в 
массиве $_SERVER[ ], его состав зависит от версии Apace, PHP,
ОС. Данные лучше уточнить при конкретной реализации.
